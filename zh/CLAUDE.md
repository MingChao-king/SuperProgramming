## Development Methodology

This plugin's development follows SuperProgramming recursively — the methodology
the plugin enforces is the same one used to build it.

### Core Principles

1. **四级缓存**: L1(CLAUD.md + SessionStart) → L2(通用经验) → L3(项目速查) → L4(源码).
2. **大纲先行**: No code without design outline → save to roadmap.
3. **TDD 三元组**: RED(test first) → GREEN(minimal impl) → REFACTOR(clean).
4. **一功能一测一审**: Each sub-function: TDD → 8-dimension subagent review → full testing.
5. **闭环**: Every task ends with roadmap update + cross-project experience extraction.
6. **防重复**: Check roadmap §4 before creating anything new.
7. **Roadmap 唯一真相源**: 6-section roadmap replaces experience.md. Design + interfaces + progress + experience all in one file.

Full methodology: use `Skill` tool → superprogramming skills.

## Anti-Duplication Checklist

Before writing ANY new class/method:

| # | Check | Command |
|---|-------|---------|
| 1 | **Roadmap §4** | Read roadmap §4 first — already indexed? |
| 2 | New Service method? | `Grep "methodName" **/service/**/*.java` |
| 3 | New DTO/VO? | `Glob **/dto/**`, `Glob **/vo/**` |
| 4 | New util method? | `Grep "methodSig" **/utils/**`, `**/common/**` |
| 5 | New Mapper method? | `Grep "methodName" **/mapper/**` |
| 6 | New constant/enum? | `Grep "VALUE_NAME" **/enums/**` |
| 7 | New dependency? | `Grep "artifactId" pom.xml` |
| 8 | Similar concept exists? | Grep business keyword |
