## Subagent Roles

3-agent pipeline per task (learned from superpowers):

| Role | Tools | Responsibility |
|------|-------|---------------|
| **Implementer** | Full | TDD RED→GREEN→REFACTOR + self-review + git commit |
| **Spec-Reviewer** | Read-only | Verify output matches task spec — no more, no less |
| **Quality-Reviewer** | Read-only | Verify code quality, patterns, DRY, security, error handling |

**Pipeline**: implementer → spec-reviewer → quality-reviewer.  
**Review Loop**: reviewer reports → implementer fixes → re-review (≤3 rounds).  
**Scout**: Lightweight read-only subagent for parallel L4 exploration, returns structured summaries.  
**Prompt Structure**: All subagents use unified 5-segment: SCOPE → GOAL → CONSTRAINTS → DO NOT → RETURN.

## Skill Writing Specification

Every SKILL.md must have:

1. **Frontmatter**: `name` and `description` (required)
2. **Skill type**: 刚性 (rigid — strict follow) or 柔性 (flexible — adapt to context)
3. **EXTREMELY-IMPORTANT block** (rigid skills): hard gate before content
4. **Red Flags table**: 3-7 diagnostic rationalizations + rebuttals
5. **完成标准 (Completion checklist)**: actionable checkboxes
6. **Mandatory language**: "必须" not "建议" (must, not should)

Skills follow superpowers design: concise, zero-dependency, pure Markdown. Target: <150 lines for rigid skills, <200 lines for complex flexible skills.
